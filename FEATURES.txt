BACKEND SERVER
================================================================================

  - Simple, lightweight design
  - No external dependencies (beyond sqlcmd)
  - CORS-enabled for extension access
  - Real-time SQL Server queries

‚úì Data Fetching
  - Comprehensive SQL query
  - Joins: GroomingLog, Pets, Clients, Employees
  - Subqueries for history and future appointments
  - Efficient single query for all data

‚úì API Endpoint
  - GET /api/waitlist
  - Returns JSON format
  - Includes metadata (count, timestamp)
  - Error handling built-in

‚úì Easy Setup
  - Double-click batch file (Windows)
  - Single command line (Linux/Mac)
  - Configurable SQL connection
  - Runs on any available port

DATA FIELDS COLLECTED
================================================================================

APPOINTMENT DATA:
- GLSeq (Appointment ID)
- GLDate (Target date)
- GLDateEntered (When added to waitlist)
- GLInTime (Requested time)
- GLDescription (Waitlist notes)

CLIENT DATA:
- CLLastName (Last name)
- CLPhone1 (Primary phone)
- CLWarning (Client warnings)

PET DATA:
- PtPetName (Pet name)
- PtWarning (Pet warnings)
- PTGroomWarning (Grooming-specific warnings)

SERVICE DATA:
- Service type (Full Service, Handstrip, Bath, Groom)
- Preferred groomer name

HISTORY DATA:
- Last completed appointment date
- Last completed appointment notes
- Next scheduled appointment date

EMPLOYEE DATA:
- Groomer first name (from GLGroomerID or GLOthersID)
